# 区块链扩容技术

## 关键技术概述

区块链扩容主要有三类思路：增加单区块容量、提高区块生成频率、以及通过架构与并行化手段提升整体吞吐量。

### 1. 区块扩容（Block Size Increase）

提高区块大小上限，从而增加单个区块可承载的交易数量，适用于短期内提升吞吐的场景，但会增加节点存储与带宽负担。

### 2. 频率扩容（Increase Block Frequency）

缩短区块生成间隔，提高单位时间内生成的区块数，能提升 TPS，但会带来更多分叉和网络同步压力。

### 3. 架构扩容（Architectural Approaches）

- 隔离见证（Segregated Witness, SegWit）
- 基于 DAG 的新型区块链架构（以 IOTA 为例）

### 4. 并行扩容（Parallel Scaling）

1. 闪电网络（Lightning Network）：链下支付通道，减轻主链负载；
2. 锲入式侧链技术（侧链/锲入机制）：将部分交易或智能合约迁移到侧链处理；
3. 二次分片（Layer-2 Sharding / 二层分片）：并行处理子集状态/交易；
4. Plasma：可扩展的链下智能合约和批量结算方案。

## 制约因素

### 网络负载

- 区块容量：更大区块需要更高带宽和更长传播时间；
- 区块频率：更高频率增加链上消息量，可能导致更多孤块/分叉。

### 节点瓶颈

- 节点性能：CPU、内存与 I/O 限制并行处理能力；
- 存储成本：长期全量存储增长导致节点运行成本上升。

## 安全问题

扩容在提高吞吐的同时会带来新的安全挑战，尤其是在高交易量或并行结构下。

### 高交易量下的协议性安全方案

#### GHOST 协议

GHOST 选择“子树工作量最大”的分支而非简单最长链，用以减少由于高出块率导致的安全折损。

#### SPECTRE 协议

SPECTRE 基于 blockDAG 的结构设计：将区块表示为有向无环图（DAG），每个区块可以引用多个先行区块（而非仅引用上一个区块）。

- 区块集合定义：对任意区块 x，Past(x) 表示可由 x 到达的前代区块集合；Future(x) 表示可到达 x 的后代区块集合，二者共同刻画因果关系。
- 分叉判定（投票规则）：对分叉区块 x 与 y，任意区块 B 按下列规则投票：
	- 若 x、y 均属于 Past(B)，则按 Past(B) 的多数派结果投票；
	- 若 x ∈ Past(B) 而 y ∉ Past(B)，则投票“x<y”；
	- 若 y ∈ Past(B) 而 x ∉ Past(B)，则投票“y<x”；
	- 若 x、y 均不属于 Past(B)，则按 Future(B) 的多数派结果投票。

该投票机制可在高并发出块场景下给出更鲁棒的最终性判定。

#### PHANTOM 协议

PHANTOM 亦为基于 DAG 的并行扩容协议，重点在于通过抽取高质量主链子图来恢复类似链式的安全性与性能权衡。

### 中心化风险

1. 算力中心化：采矿/出块集中化会削弱去中心化安全属性；
2. 节点中心化：高硬件门槛导致节点数量下降；
3. 客户端中心化：轻客户端或第三方服务集中化风险。

### 侧链与跨链安全性

侧链引入跨链桥与铸销机制，需关注跨链验证、资产锁定与欺诈证明等安全问题。

## 经济问题

- 币值：扩容对供需、交易确认体验与币价波动具有间接影响；
- 交易费用与矿工收益：不同扩容方案会改变手续费市场与矿工/验证者的收益模型。
