# 第10章：DAG区块链项目

## IOTA（Tangle）

### 数据结构

IOTA 使用 Tangle DAG：

- 每一笔新交易 = DAG 中一个节点
- 每笔新交易必须 验证之前的两笔交易
- 没有区块、没有矿工

结构示意：

```
tx1 ← tx3 ← tx6
tx2 ← tx4 ← tx7
      ↘ tx5 ← tx8
```

### 共识机制

核心：基于概率权重的累计确认

- 每个交易都有 weight
- 后续交易越多引用你，你的 cumulative weight 越大
- 当权重大于阈值 → 视为最终确认

采用 MCMC 随机游走算法 选择要验证的两笔交易。

### 交易确认流程

- 发起交易
- 随机挑选 2 笔未确认交易进行校验
- 做少量 PoW 防止垃圾交易
- 广播给网络

你想转账，必须先帮别人"打工验证"。

### 特点与问题

| 项目     | 说明                     |
|----------|--------------------------|
| 吞吐量   | 理论上 TPS ∝ 网络活跃度 |
| 手续费   | 0                        |
| 适合     | IoT 微支付               |
| 问题     | 早期依赖 Coordinator（中心节点），去中心化不足 |
| 现状     | Coordicide 正在逐步移除 |

## Byteball（现 Obyte）

### 数据结构

仍是 DAG，但引入 主链（Main Chain）概念

通过 Witness（见证人）列表 排序交易

每个交易：

- hash -> 父交易集合

### 共识机制：见证人 + 确定性排序

网络中固定一组 12 位 Witness：

- 交易被最多 Witness 间接引用
- 通过 Witness 顺序形成主链
- 主链用于 解决冲突与双花

不是投票，而是结构确定性共识。

### 交易确认流程

- 提交交易
- 被 Witness 引用并进入主链
- 一旦主链深度达到阈值 → 最终确认

### 特点与问题

| 项目     | 说明                     |
|----------|--------------------------|
| TPS      | 远高于区块链           |
| 延迟     | 秒级                     |
| 去中心化 | 依赖 Witness 列表（半中心化） |
| 创新点   | 稳定币、条件支付、Oracle 内置 |
| 问题     | Witness 改选难、参与门槛高 |

## Hashgraph（最学术、最强性能）

### 数据结构

DAG 中的节点叫 Event

每个 Event 包含：

- 交易数据
- 父节点 Hash（self-parent & other-parent）

```
E1 → E3 → E5
  ↘ E4 → E6
E2 ↗
```

### 共识机制：Gossip about Gossip + 虚拟投票

这是核心创新。

#### Gossip about Gossip

节点每次 Gossip 同时发送：

- 交易内容
- 自己的通信历史（即 DAG 结构）

于是每个节点最终都拥有全网一致的 DAG 拓扑图。

#### 虚拟投票（Virtual Voting）

因为所有节点 DAG 一致：

- 无需真正投票
- 本地即可计算出：
  - 谁投了什么票
  - 谁赢得了共识

数学上直接推导。

### 共识属性

- aBFT（异步拜占庭容错）
- 不需要 leader
- 不存在分叉

### 性能与问题

| 项目     | 说明                     |
|----------|--------------------------|
| TPS      | 10W+                     |
| 延迟     | 3~5 秒                   |
| 共识安全 | aBFT 理论最强           |
| 问题     | 专利闭源，企业许可制   |
| 去中心化 | 权限网络为主           |

## 三者本质对比

| 维度     | IOTA      | Byteball   | Hashgraph   |
|----------|-----------|------------|-------------|
| DAG结构  | 无主链随机引用 | 主链 + Witness | Gossip DAG |
| 共识方式 | 概率权重 | 结构确定性 | 虚拟投票   |
| 去中心化 | 中（早期低） | 中低       | 权限制     |
| TPS      | 随负载增长 | 高         | 极高       |
| 是否公链 | 是        | 是         | 否（联盟链） |

## 技术路线总结一句话

- IOTA： 用"交易即验证"实现自激励 DAG
- Byteball： 用"见证人排序"解决 DAG 无序问题
- Hashgraph： 用"全网 DAG 共识 + 虚拟投票"实现理论最优 BFT
